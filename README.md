# Horse_Sim_Force

東京競馬場 芝2400mを舞台にした、物理法則に基づく競馬レースのエージェントベースシミュレーション（ABM）です。各馬の挙動を、恣意的なルールベースの分岐ではなく、推進力・抗力・摩擦・重力といった最小限の力学量から導出することを目指しています。

horse_sim_force.py：レースをアニメーションでシミュレーションできる
race_core.py,batch_runner.py：レースをアニメーション抜きでシミュレーションし、複数回回してデータを収集できる

実行例：両ファイルを同じフォルダで以下のように実行してください
python batch_runner.py --races 500 --equalize --jobs 4

## 概要

18頭の馬をそれぞれ独立したエージェントとして扱い、各ステップで力のつり合いから加速度を計算し、位置と速度を更新していくシミュレーションです。コースの形状、勾配、馬体重、空気抵抗、コーナリングの限界速度などをすべて物理量として扱い、レース展開をパラメータの恣意的な調整ではなく力学的な相互作用から自然に生み出すことを設計上の目標としています。

## 設計思想

本モデルは以下の力学要素から構成されています。

- **推進力**：目標速度への緩和過程として表現し、応答時間（TAU）によって反応の速さを調整しています。
- **空気抵抗とドラフティング**：後方かつ近い角度に他馬がいる場合、空気抵抗係数を動的に低下させ、風よけ効果を再現しています。ドラフティングで得たエネルギーはスパートのタイミング判定に利用されます。
- **コーナリング限界速度**：摩擦係数と重力加速度から `v = sqrt(μgr)` として算出し、物理的に無理のない速度でコーナーを通過させています。
- **坂道の影響**：勾配に応じた重力成分を推進力に加算し、上り坂での失速・下り坂での加速を再現しています。
- **衝突回避とレーン取り**：反発力による衝突回避と、前を走る馬との位置関係に応じたステアリング制御によって、進路取りや追い抜きの挙動を表現しています。
- **動的なスパート戦略**：残り距離と貯蓄エネルギーの閾値判定によってスパート開始タイミングが決まり、終盤は貯蓄エネルギーに応じた疲労による減速が発生します。

## コース設計

東京競馬場 芝2400mの実測値を再現しています。

- 1周距離：2083.1m
- ゴール前直線：525.9m
- コーナー半径：上記の数値から逆算して算出
- ホームストレッチとバックストレッチの高低差を勾配として反映

出典:JRA公式サイト(https://www.jra.go.jp/facilities/race/tokyo/course/)  

## 主な機能

- コース勾配のプレビュー表示（色の濃淡で上り・下りを可視化）
- レース展開のリアルタイムアニメーション
- レース結果の表形式出力（順位、タイム、着差、各コーナー通過順位、上り3Fタイム）

## 使用技術

- Python 3
- NumPy
- Matplotlib

## 実行方法

```bash
pip install numpy matplotlib
python horse_sim_force.py
```

実行すると、まずコース勾配のプレビューが表示され、ウィンドウを閉じるとレースアニメーションが開始します。レース終了後、結果が表形式でコンソールに出力されます。
